{% docs event_id %}
Unique identifier for each event in the system. Generated by the source system at event creation time. Used as the primary key for event-related tables.

**Format**: `evt-XXX` where XXX is a sequential number
**Example**: `evt-001`, `evt-002`
{% enddocs %}

{% docs user_id %}
Unique identifier for each user in the system. Links events to the user who performed them. Used as the primary key in user dimension and foreign key in event facts.

**Format**: `usr-XXX` where XXX is a sequential number
**Example**: `usr-101`, `usr-102`
{% enddocs %}

{% docs event_type %}
Categorizes the type of user interaction or system event. Used for filtering and aggregating events by behavior type.

**Valid Values**:
- `page_view` - User viewed a page
- `click` - User clicked an element
- `purchase` - User completed a purchase
- `signup` - New user registration
- `login` - User authentication
- `logout` - User session end
{% enddocs %}

{% docs event_timestamp %}
The exact moment when the event occurred, captured in UTC timezone. Used for time-series analysis and event ordering.

**Format**: ISO 8601 timestamp
**Example**: `2024-02-23T10:00:00Z`
{% enddocs %}

{% docs event_date %}
Date portion of the event timestamp, extracted for partitioning purposes. Iceberg tables are partitioned by this field for efficient date-range queries.

**Format**: `YYYY-MM-DD`
**Example**: `2024-02-23`
{% enddocs %}

{% docs username %}
Display name chosen by the user. May contain spaces and special characters. Used for human-readable identification in reports and dashboards.

**Example**: `Alice Smith`, `Bob Johnson`
{% enddocs %}

{% docs email %}
User's email address. Must be unique across all users. Used for user identification and communication.

**Format**: Standard email format
**Example**: `user@example.com`
{% enddocs %}

{% docs is_active %}
Boolean flag indicating whether the user account is currently active. Inactive users may have been deactivated by admins or through account deletion requests.

**Values**: `true` (active), `false` (inactive)
{% enddocs %}

{% docs created_at %}
Timestamp when the record was first created in the source system. Used for tracking data lineage and identifying new records.

**Format**: ISO 8601 timestamp
{% enddocs %}

{% docs updated_at %}
Timestamp when the record was last modified in the source system. Used for incremental processing and change data capture.

**Format**: ISO 8601 timestamp
{% enddocs %}

{% docs country %}
Two-letter ISO country code representing the user's country of residence or registration.

**Format**: ISO 3166-1 alpha-2
**Examples**: `US`, `CA`, `UK`, `DE`
{% enddocs %}

{% docs session_id %}
Unique identifier for a user session. Groups related events that occurred during a single user visit.

**Format**: `sess-XXX`
**Example**: `sess-001`
{% enddocs %}

{% docs amount %}
Monetary value associated with purchase events. Represents the transaction amount in USD.

**Format**: Decimal number
**Example**: `99.99`, `149.50`
{% enddocs %}

{% docs page %}
URL path of the page where the event occurred. Used for page-level analytics and user journey analysis.

**Example**: `/home`, `/products`, `/checkout`
{% enddocs %}
